Signal chain

      │                                                              │

  • Wired output only — never Bluetooth. A2DP re-encodes through a lossy codec that bakes an uncontrolled EQ and time-smear into every trace. Latency is survivable; codec colouration is fatal.
  • Line-level input only (Aux / CD / Tape). Never phono/turntable — it applies RIAA EQ and wrecks the signal.
  • iPhone 15+ : USB-C DAC → 3.5mm-to-RCA → line in. iPhone 14− : Lightning-to-3.5mm → RCA.
  • Mic = the device's own built-in mic, at the listening position (or near the cone for Nearfield / bass work).

Validity & the honesty caveats

  • This is a relative bench, not a calibrated SPL meter. No absolute FR, no SPL, no impedance. Mic error is a magnitude error — so timing/phase and A/B comparisons are immune.
  • Biggest risk: Safari/Chrome may quietly re-apply AGC / noise-suppression / echo-cancellation despite our constraints. Run the Repeatability check — two back-to-back sweeps should overlay within ~1 dB. If they don't, the input is being processed and the numbers can't be trusted.
  • Gate length sets the low-frequency limit: valid above ≈ 1000 / (gate-post ms) Hz. For bass, use Nearfield mode (mic at the cone) to dodge the room.

Mic calibration & using a UMIK

  • The phone/laptop mic is uncalibrated — its error is a magnitude error, so timing, phase, driver-offset and A/B comparisons are already valid. What's NOT valid is the absolute shape of the frequency response.
  • Load a calibration file (miniDSP UMIK .txt, or REW/generic .frd/.cal) on the Measure tab to correct the magnitude shape. With a UMIK-1 plugged into a laptop's USB and selected as the Input device, you get a genuinely calibrated relative frequency response — the kind that drops into VituixCAD.
  • For VituixCAD, the hard part is timing between drivers, not magnitude. A UMIK is a single USB channel, so it can't do a dual-channel loopback timing reference. Use the Driver-Offset mode here to get the acoustic z-offset in mm directly — that's the number VituixCAD asks for and most people just guess. Measure each driver from a fixed mic position and don't move the mic between captures.
  • Absolute SPL is still out of scope. The UMIK's Sens Factor is read and shown, but a browser can't reliably know the input gain chain, so this stays a calibrated-relative bench. For true SPL, use REW on a computer.

About REWMitch — how it works, and its limits

REWMitch is an on-device speaker-measurement bench: it plays an exponential sine sweep, records the room/driver on the device mic, and derives the impulse response, frequency response, phase, group delay, driver time-offset, harmonic distortion and a decay waterfall — all client-side, nothing uploaded.

The one idea everything rests on: the device mic is uncalibrated, but its error is a magnitude error. So anything time- or phase-based, and anything done as an A/B comparison at one fixed mic position, is unaffected by the mic. REWMitch is therefore a relative & timing bench, not a calibrated SPL meter — and it is designed honestly around that.

Reviewer questions — answered directly

Specific points raised by an experienced tester. Thank you — this is exactly the scrutiny that makes the tool honest.

Q. REW has "Adjust clock with acoustic/loopback ref" — a second timing marker at the end of the sweep to compensate drift between async capture/playback clocks, verified by running two sweep lengths and checking the relative phase matches. Do you do this?

A. Yes — REWMitch does exactly this (since v2.0). It plays a timing marker at the start AND end of the sweep, measures how far the marker gap stretched in the recording (the clock ratio, in ppm), and resamples the recording back to the playback clock before deconvolution. It's a no-op when clocks are shared (deadband) and rejects implausible ratios. Your exact verification method is built in: a dev self-test (rewmitchDriftSelfTest() in the browser console) runs the same fixed setup at two sweep lengths (1 s and 4 s) and reports the group-delay RMS difference with compensation ON vs OFF — ON should collapse to a few µs; OFF should visibly diverge, and grow toward HF and with the longer sweep. The Standard-mode status line reports "clock drift corrected +X ppm" or "clocks in sync." Ref: REW Help — acoustic/loopback timing reference; AVNirvana forum.

Q. I can hear the markers and sweep and see the input meter, but I don't get any magnitude or phase graph.

A. Two things — and the first is that you found a real bug, now fixed (v10). The Measure-tab plot canvas was created while its tab was hidden at page load, so it stayed zero-sized and Standard-mode graphs rendered blank. It now re-measures itself whenever the tab is shown or a trace is drawn, so the plots appear. Genuinely — thank you for surfacing it. Second, if you happened to be in Driver-Offset mode, that one is timing-only by design: it outputs a single z-offset in millimetres (after you capture both drivers and press Compute offset), not a graph — the markers you hear are its acoustic reference. Magnitude / phase / group-delay curves live on the Measure tab → Standard mode (Magnitude / Phase / Group-delay toggles above the plot); the wizard's Relative phase test overlays the two drivers' phase. (Also fair that the beginner "Guide" tab is the default landing — an expert wants the Measure tab.)

Q. I hear the markers and sweep from BOTH L and R. The acoustic reference should use one channel for the speaker under test and the other for a (HF-capable) timing-reference speaker.

A. Correct — and there are two different "reference" mechanisms, so which applies depends on the mode:
• Driver-Offset (true two-channel acoustic reference): the timing marker goes to the reference channel (your fixed, HF-capable other speaker — it must reproduce the 2–4 kHz marker) and the sweep to the test channel; they're separate channels, and the reference channel is L/R-selectable in the offset panel. If you hear both signals from both sides, your output is summing to mono (single speaker, mono DAC, or the DUT wired to both channels) — the offset needs true stereo with the reference speaker on the opposite channel.
• Clock-drift (Standard/linear): here the two drift markers ride on the same channel as the sweep — they only need to share the play/record clock, not a separate speaker — so you'll hear them from the measured speaker (and from both sides, because that path is mono). That's expected. A REW-style separate reference-channel option for the linear path is a sensible future addition. Ref: REW Help; AVNirvana forum.

Q. There's a mic calibration file but no soundcard/DAC calibration file.

A. True — mic cal is supported, output/soundcard cal isn't (yet). Reasoning: a competent USB DAC/dongle is essentially ruler-flat across the audio band, and whatever it does is identical in every capture, so it cancels in all relative, A/B and driver-offset (timing) results — which is what REWMitch is for. It would only bias the absolute magnitude shape, which we already don't claim. But it's a fair "nice to have," and it's easy (same mechanism as the mic cal — load an .frd/.txt and subtract), so it's on the shortlist.

Q. For the mic calibration file — are you ignoring the phase and taking only the magnitude of the mic response?

A. Yes, exactly. The parser reads the phase column if the file has one, but only the magnitude correction is applied; the phase is not. Rationale: the common files (miniDSP UMIK) are magnitude-only anyway, a measurement mic's own phase error is small, and — being a fixed characteristic of the mic — it cancels in relative/A-B and in driver-offset timing (both drivers see the same mic). It would matter only for absolute single-measurement phase; applying it is straightforward if your cal file genuinely carries phase — say the word and we'll enable it.

Q. What time-gating window function does the tool use (rectangular / Hamming / Hann / Blackman-Harris…)?

A. A Tukey window (cosine-tapered rectangular) — not a plain Hann or Blackman-Harris over the whole gate. Concretely: a flat (rectangular, gain 1.0) centre with a raised-cosine (Hann) half-taper on each edge; the taper lengths equal the pre-gate (default 1 ms) and post-gate (default 5 ms). Why: the flat centre preserves the direct-sound energy right at the impulse peak (a full-length Hann/Blackman-Harris would attenuate the peak itself and bias level/timing), while the cosine edges avoid the spectral leakage a hard rectangular truncation would cause. Ref: F. J. Harris, "On the Use of Windows for Harmonic Analysis with the Discrete Fourier Transform," Proc. IEEE, 1978; REW Help — measurement windows.

Strengths

  • Driver time-offset (the headline). Pure time-of-flight, zero calibration needed. Self-referenced against a fixed reference speaker so play/record latency and clock jitter cancel. Reports the acoustic z-offset in mm with an honest ± error bar.
  • Relative phase & group delay between two drivers at one mic position — the crossover-summing question — is immune to the mic's magnitude error.
  • Before/after & A/B magnitude. The difference between two traces (a stuffing change, a resistor, a port bung) is trustworthy even though the absolute curve isn't.
  • Gross distortion (breakup, buzzes, rattles) via the log-sweep harmonic method.
  • Clock-drift compensation for separate play/record clocks (USB DAC out + phone mic in), which otherwise smears HF phase.
  • Runs anywhere, offline, installs as an app, no account, no data leaves the device.

Limitations (read these)

  • No absolute SPL and no calibrated absolute frequency-response shape from the built-in mic. Load a mic calibration file (UMIK/REW) to fix the shape; absolute SPL is still out of scope (see Q&A).
  • Gated far-field FR is only valid above ≈ 1000 / (gate-post ms) Hz (default gate → ~200 Hz). Below that, use Nearfield mode, which shows shape, not level.
  • Distortion is limited by the mic's own distortion floor — good for gross faults, not subtle THD figures.
  • The browser may quietly re-apply AGC / noise-suppression / echo-cancellation. Always run the Repeatability check first (two sweeps should overlay within ~1 dB).
  • Not a crossover simulator. REWMitch measures; tools like VituixCAD do the driver summation, diffraction and merging.

Q&A (for the technically minded)

Q. Which deconvolution method — division or the matched/inverse filter?

A. Two, deliberately. For FR/phase/group delay it's frequency-domain regularised division (IR = Rec·conj(S)/(|S|²+ε)), because it's latency-immune and needs no harmonic separation. For distortion it's Farina's time-reversed inverse filter, because that's what makes the harmonics fall into the pre-impulse region so they can be isolated [Farina 2000].

Q. Do you window the impulse response? With what?

A. Yes — a raised-cosine gate, 1 ms before / 5 ms after the peak by default (wider in Nearfield). That's a single, fixed, transparent gate. It trades low-frequency resolution for room-reflection rejection: valid above ≈ 1000/(post-ms) Hz, shown on screen [Müller & Massarani 2001; REW Help].

Q. Do you use frequency-dependent (psychoacoustic) windowing, like REW's FDW?

A. No — see "what we deliberately don't do" below.

Q. Is the phase minimum-phase, excess-phase, or measured?

A. Measured (total) phase, straight from the complex spectrum, plus group delay. We don't split out minimum-phase/excess-phase (Hilbert) — again, see below.

Q. What smoothing, exactly?

A. Fractional-octave, averaged in the power domain (not dB), selectable 1/3, 1/6 (default), 1/12, or off. No variable/psychoacoustic ("Var") smoothing.

Q. What speed of sound do you assume for the offset-to-mm conversion?

A. A fixed 343 m/s (≈20 °C). The true value is c ≈ 331.3 + 0.606·T(°C) m/s, so the error is ≈ 0.17 %/°C — e.g. at 26 °C you'd read ~1 % high. For a 20 mm offset that's ~0.2 mm, well under the measurement's own error bar, so it isn't user-settable (yet). Humidity's effect is far smaller again.

Q. How is the offset made immune to latency/clock jitter?

A. Each capture contains both the acoustic reference marker and the driver's sweep, so both share the same play/record delay; subtracting the two arrivals cancels it. The final z = (drvB − refB) − (drvA − refA) also cancels the reference-speaker distance — it need only stay constant. Peak timing is sub-sample (parabolic on the cross-correlation / IR peak).

Q. Why not a loopback/electrical timing reference?

A. Phones/tablets have one input channel, so a dual-channel electrical loopback isn't possible. The acoustic marker is the single-channel equivalent, and for clock drift a second marker at the end of the sweep measures the timebase stretch directly [REW acoustic timing reference].

Q. How trustworthy is the distortion?

A. For gross faults, good; for absolute THD numbers, treat as indicative. It's bounded by the device mic's own distortion, which is typically far worse than a good driver's [Farina 2000].

What we deliberately DON'T do (and why)

Each of these was a conscious call, with the trade-off stated and a reputable reference to check it against.

  • Absolute SPL. A browser hands us normalised samples with an unknown input gain, and the built-in mic has no sensitivity reference. Real SPL needs a calibrated mic + a known reference level (e.g. 94 dB @ 1 kHz) or a UMIK Sens-Factor pipeline. Ref: REW Help — SPL calibration; miniDSP UMIK-1 (minidsp.com).
  • Frequency-dependent / psychoacoustic windowing (FDW). A single fixed gate is transparent and predictable; FDW is powerful but can hide or invent detail if misapplied, and adds a per-frequency window we didn't want to hide behind. If you need it, REW does it well. Ref: REW Help — "Frequency Dependent Window".
  • Minimum-phase / excess-phase separation. We show the measured phase and group delay as-is rather than deriving a minimum-phase model (Hilbert transform) — fewer hidden assumptions for a relative bench. Ref: REW Help — minimum phase; Müller & Massarani 2001.
  • Nearfield-to-farfield scaling, baffle-step / diffraction correction, and driver merging/summation. That's the crossover-simulator's job; REWMitch is the measurement front-end that feeds it. Ref: VituixCAD (kimmosaunisto.net); Keele, "Low-Frequency Loudspeaker Assessment by Nearfield Sound-Pressure Measurement", JAES 1974.
  • Impedance & Thiele–Small parameters. That's an electrical measurement (sense resistor + current sensing), out of scope for an acoustic-only tool. Ref: REW impedance measurement; miniDSP DATS.
  • Bluetooth output. A2DP re-encodes through a lossy codec that bakes in its own EQ and time smear which can't be separated from the speaker. Wired only. Ref: REW Help — measurement setup.
  • Spatial / moving-mic averaging (MMM). Single fixed-position captures only, to keep timing/phase meaningful. Ref: REW Help — MMM.
  • Soundcard / output (DAC) calibration file. Only the mic is calibratable. A competent USB DAC is essentially flat and, being identical in every capture, cancels in all relative/A-B/timing results — so it only affects the absolute magnitude shape we don't claim. On the shortlist as a "nice to have." Ref: REW Help — soundcard calibration.
